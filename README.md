## ことひろのフロント実装

### 🌐 プレビュー環境

各ブランチごとに異なる環境が用意されており、自動デプロイされます。

| ブランチ名 | 環境           | ドメイン             | 用途                         |
| ---------- | -------------- | -------------------- | ---------------------------- |
| `develop`  | 開発           | develop.kotohiro.com | 開発中の機能のプレビュー確認 |
| `staging`  | ステージング   | staging.kotohiro.com | リリース前の最終確認環境     |
| `main`     | プロダクション | kotohiro.com         | 本番環境                     |

### 🔄 デプロイフロー

1. **開発環境 (`develop`)**

   - 開発中の機能を確認するための環境
   - PRマージ後に自動デプロイ

2. **ステージング環境 (`staging`)**

   - QAやテストを行うための環境
   - `develop`ブランチからのPRマージ後に自動デプロイ

3. **本番環境 (`main`)**
   - エンドユーザー向けの本番環境
   - `staging`ブランチからのPRマージ後に自動デプロイ

## 🚀 開発の始め方

### 必要な環境

- Bun (パッケージマネージャー)

### 開発環境のセットアップ

```bash
# 依存関係のインストール
bun install

# 開発サーバーの起動
bun run dev

# 型チェックの実行
bun run typecheck
```

### コード品質管理

```bash
# コードフォーマット
bun run format

# リントチェック
bun run lint
```

### UIコンポーネント開発

```bash
# Storybookの起動
bun run storybook

# SVGアイコンのコンポーネント生成
bun run icongen
```

```bash
# プレビュー環境へのデプロイ
bun run deploy
```
